# ReadAloud 新首页与主工程整合说明

> **已合并（ReadAloud 2.0）**：新首页代码已迁入仓库根目录 `src/`，原 `readaloud-home-ui/` 子目录已删除。下文中的路径与阶段划分保留作历史记录；当前以根目录 `src/` 与 `vite.config.ts` 为准。

本文档曾约定 **新首页（`readaloud-home-ui`）** 与 **主应用（仓库根目录 `src/`）** 的协作方式。

---

## 目标

- 在新首页上继续增加 **交互**、接入 **原有能力**（TTS、翻译、查词等），并开发 **新功能与新界面**。
- **默认不破坏**主工程现有行为；合并进主应用时 **可开关、可回滚**。

---

## 阶段划分

### 阶段 A — 子项目内迭代（当前主战场）

- 路径：`readaloud-home-ui/`
- 在此完成：交互、路由骨架、占位与假数据、视觉与布局调优。
- **不要**把主工程里大段业务代码复制进子项目，避免两套逻辑长期分叉。

### 阶段 B — 抽出共享能力（`packages/shared` 或等价命名）

- 将主工程中可复用的 **纯逻辑** 抽到独立包，例如：

  - TTS 调用与封装
  - `fetchApi`、与后端的请求约定
  - 词典 / 翻译等 **不依赖 React 组件** 的模块

- 仓库根目录使用 **npm workspaces**（或 pnpm workspace），例如：

  - `packages/shared` — 被主应用与 `readaloud-home-ui` 共同依赖
  - 主应用原有页面 **逐步** 改为从 `@readaloud/shared`（或实际包名）引用，而不是一次性大改。

- **原则**：共享包内尽量 **不依赖 React**；若必须，需统一版本，避免双 React 实例问题。

### 阶段 C — 新首页并入主应用

- 在新 UI 稳定后，在主工程中 **增量** 接入：

  - 增加新路由（例如 `/readaloud/new-home`），或
  - 使用环境变量 / 功能开关（如 `VITE_USE_NEW_HOME`）在默认首页与新版首页之间切换。

- 旧入口保留一段时间，验证无误后再替换默认首页或下线旧界面。

## 如何减少对「原项目文件」的影响

| 策略 | 说明 |
|------|------|
| 子项目独立开发 | 合并前，实验与迭代主要在 `readaloud-home-ui/`，主工程 `src/app/App.tsx` 等可 **暂不修改**。 |
| 共享逻辑用包，禁止复制粘贴 | 不在子项目里维护第二份 `tts.ts`；统一进 `packages/shared`，修 bug 只改一处。 |
| 主应用增量接入 | 新目录（如 `src/app/home-v2/`）与旧代码并列；通过路由或开关切换，避免大面积删除。 |
| Git 分支 | 使用 `feature/new-home` 等分支承载新首页与 shared 抽取；`main` 保持稳定，合并前 Review。 |

## 技术注意点

1. **两套构建**  
   主应用与 `readaloud-home-ui` 各自有 Vite/React，版本可能不同。`packages/shared` 以 **纯 TS + 可选类型** 为主，减少对 React 的依赖。

2. **API 与路径**  
   主应用可能使用 `base: "/readaloud/"` 与 `/api` 代理。子项目联调时：  
   - 要么 **同源部署** 后联调；  
   - 要么在 `readaloud-home-ui/vite.config.ts` 中 **proxy** 到与主应用一致的后端，避免路径不一致。

3. **环境变量**  
   密钥与 API 基地址使用 `.env`；shared 内若需要 URL，由调用方注入 `baseURL` 或从各自 `import.meta.env` 读入，避免在包内写死环境。

4. **键盘与单屏**  
   新首页已做的 viewport / `dvh` / 输入框字号等约定，若在主应用挂载，需在 **同一套 meta 与根布局** 下复测移动端。

## 推荐执行顺序（ checklist ）

1. 在 **`readaloud-home-ui`** 完成交互与导航、空状态与主要用户路径。  
2. 梳理主工程待复用模块 → 建立 **`packages/shared`**（或选定命名）并迁入。  
3. 主应用 **小范围** 验证 shared（例如仅一处按钮调用 TTS）。  
4. 新首页以 **新路由或开关** 并进主应用，观测无误后再切换默认入口。
